# Method + Engine Implementation Status

## Overview

This document tracks which method+engine combinations are implemented for TurboQuantProdFactory.

### Available Methods
- **planarquant**: 2D Givens rotation (fastest, by ParaMind2025)
- **isoquant**: 4D quaternion rotation (recommended default)
- **rotorquant**: Clifford 3D rotation (legacy)
- **turboquant**: Dense d×d rotation (Google reference)

### Available Engines
| Engine | Backend | Device | Description |
|--------|---------|--------|-------------|
| cpu | python | cpu | Pure Python/PyTorch on CPU (no GPU) |
| torch_cpu | python | cpu | PyTorch operations on CPU |
| cuda_kernel | cuda | cuda | Compiled CUDA .so kernels |
| torch_cuda | python | cuda | PyTorch operations on CUDA |
| triton | triton | cuda | Triton GPU kernels |

## Missing Implementations Status

### 1. isoquant + cuda_kernel
- **Status**: Needs C++ wrapper development (NOT quick Python work)
- **Kernel exists**: `methods/isoquant/cuda_kernels/isoclinic_fused.so`
- **Issue**: No Python wrapper that loads and uses this .so
- **Work required**: Create C++/Python bridge or use ctypes

### 2. rotorquant + triton  
- **Status**: NEEDS PYTHON WRAPPER
- **Kernels exist**: `methods/rotorquant/rotorquant_triton.py`
- **Fix**: Add class wrappers (IsoQuantMSE-style) and register in factory
- **Work required**: ~50 lines of Python wrapper code + registration

### 3. turboquant + triton
- **Status**: NOT IMPLEMENTED
- **Work required**: Full implementation of dense rotation in Triton

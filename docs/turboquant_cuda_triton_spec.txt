# TurboQuant 5-Engine Implementation Spec

This document records the instance specifications obtained from benchmarking and code analysis for implementing TurboQuant's 5 engines: cpu (numpy), torch_cpu, torch_cuda, cuda_kernel, and triton.

## Architecture Overview

### Paradigm Shift: QJL Removal from KV Cache

**Critical Discovery**: QJL (1-bit quantized Johnson-Lindenstrauss transform) is **NOT suitable for KV Cache** despite being part of original TurboQuant design.

**Why QJL Fails in KV Cache**:

```
Problem: QJL provides unbiased inner products, but attention uses softmax
- Softmax exponentially amplifies variance
- QJL's random noise gets exponentially amplified
- MSE-only has biased inner products but LOWER variance
- Lower variance → better softmax results

Measurements:
- QJL mode: +300% attention error vs MSE-only
- MSE-only mode: +7.6% error (GPT-2 baseline)
- Generation test: QJL = 0/27 pass, MSE-only = 18/18 pass
- Qwen2.5-3B 3-bit PPL: RefTQ(QJL) = 5,128 vs IsoQuant-MSE = 170
```

**When QJL Works**: Vector search (no softmax), non-softmax attention (sigmoid, linear, gated)

**When QJL Fails**: Transformer attention with softmax (KV cache)

## KV Cache Optimization Spec

### Critical: QJL Removal Rationale

**6 Independent Teams Confirmed**: QJL + softmax = catastrophic failure

```
QJL Mechanism:
  - Adds 1-bit residual correction for unbiased inner products
  - Works great for vector search (no softmax)
  - Fails for attention (softmax amplifies noise)

Softmax Variance Amplification:
  softmax(x + noise) ≈ exp(x + noise) / Σexp(x + noise)
  Variance of exp(x + noise) >> Variance of exp(x)
  
MSE-only Advantage:
  - Biased but low-variance reconstruction
  - Softmax preserves directional information better
  - Lower PPL degradation (+7.6% vs +300%)
```

### Asymmetric K/V Bit Allocation

**Default Configuration**:
```python
TurboQuantKVCache(
    d_key=128, d_value=128,
    key_bits=4, value_bits=2,  # Asymmetric
    seed=42
)
```

**Why Asymmetric?**

| Component | Role | Sensitivity | Recommended Bits |
|-----------|------|-------------|------------------|
| Key | Q @ K^T (attention scores) | High | 4 bits |
| Value | softmax(QK^T) @ V (weighted sum) | Medium | 2 bits |

**Memory Calculation** (seq_len=100, d=128, key_bits=4, value_bits=2):
```
Key indices: 100 × 128 = 12,800 indices
Key bits: 12,800 × 4 = 51,200 bits

Value indices: 100 × 128 = 12,800 indices  
Value bits: 12,800 × 2 = 25,600 bits

Total: 76,800 bits (5.33x compression vs FP16)
FP16 baseline: 100 × 256 × 16 = 409,600 bits
```

**Implementation** (5 backends: torch_cpu, numpy, cuda_kernel, triton, + CUDA):
```python
def memory_usage_bits(self) -> dict:
    key_indices = sum(c["indices"]["indices"].numel() for c in self.key_cache)
    value_indices = sum(c["indices"]["indices"].numel() for c in self.value_cache)
    total_bits = key_indices * self.key_bits + value_indices * self.value_bits
    fp16_bits = (key_indices + value_indices) * 16
    compression_ratio = fp16_bits / total_bits
```

### Bit-Packed Storage

**Compression Ratio** (d=128, bits=3):
- Original: 128 × 2 bytes = 256 bytes/vector
- Compressed: (128 × 3 / 8) + 2 = 50 bytes/vector
- **Ratio: 5.1x compression**

**Storage Format**:
```python
{
    "indices": (B, H, S, D) uint8,    # Lloyd-Max quantized indices
    "norms": (B, H, S) float16,       # Vector L2 norms
    "shape": (B, H, S, D)
}
```

### Layer-Adaptive Bit Allocation

**Sensitive Layers Priority**:
1. Embedding layer (first)
2. First attention layer
3. Last attention layer
4. Output layer

**Configuration Example**:
```python
# Protected layers get +2 bits
TurboQuantKVCache(protected_layers=4)  # First 4 + last 4 layers
```

### GQA (Grouped Query Attention) Support

**Runtime Configuration**:
- Parameters from model config: `num_query_heads`, `num_kv_heads`
- Group size: `gqa_group_size = num_query_heads / num_kv_heads`
- Implementation: Runtime-detectable (not hardcoded)

## Interface Changes (V3)

### Before (V2 - QJL Included)
```python
TurboQuantKVCache(d_key, d_value, bits=3, seed=42)
# Stores: indices + qjl_signs + residual_norm
```

### After (V3 - QJL Removed)
```python
TurboQuantKVCache(
    d_key, d_value, 
    bits=3, 
    key_bits=None,  # Defaults to bits (4)
    value_bits=None,  # Defaults to bits (2)
    seed=42
)
# Stores: indices only (no QJL fields)
```

### Key Changes:
1. **Removed QJL fields**: `qjl_signs`, `residual_norm`, `mse_indices`
2. **Added asymmetric bits**: `key_bits`, `value_bits` parameters
3. **Interface inheritance**: All backends inherit `KVCacheBase`
4. **Memory calculation**: Separated key/value indices
5. **Forward method**: Added to avoid double dequantization

# TurboQuant CUDA 커널 로드 실패 문제 해결

## 문제 설명

### 현재 상황
- `methods/turboquant/cuda_kernels/` 에 빌드된 `.so` 파일이 존재
- `turboquant_cuda_kernel.py` 가 모듈 레벨에서 `import torch` 실행
- CUDA 라이브러리를 로드하기 전에 `import torch` 가 먼저 실행됨

### 문제 증상
1. `turboquant_factory.py` 의 `ensure_cuda_kernels_built()` 가 호출될 때마다 빌드 시도
2. 빌드 실패 (linker error: `libm.so.6 not found`, `libmvec.so.1 not found`)
3. `_CUDA_AVAILABLE = False` 로 설정
4. `experiments/turboquant_facade.py` 실행 시:
   - `mse_ratio_*` 열에만 fallback 값 표시
   - 다른 모든 열에서 `cuda_kernel` 값이 NA

### 근본 원인
**LD_LIBRARY_PATH 설정 타이밍 문제:**

`turboquant_cuda_kernel.py` 의 현재 구조:
```python
import torch  # ← 라인 14: 모듈 레벨에서 먼저 실행됨
import os

# LD_LIBRARY_PATH 설정 (라인 31-58):
#   이 시점에는 이미 torch 가 로드되어 있음
#   torch 가 CUDA 라이브러리를 찾을 수 없음
```

**문제 흐름:**
1. `import torch` 실행 → PyTorch가 CUDA 라이브러리를 로드 시도
2. `LD_LIBRARY_PATH` 설정 → 너무 늦음 (torch 가 이미 로드됨)
3. `libc10.so`, `libm.so.6` 등을 찾지 못함
4. 모듈 로드 실패 → `_CUDA_AVAILABLE = False`

**빌드 실패 원인:**
```
/data/anaconda3/compiler_compat/ld: /lib/x86_64-linux-gnu/libm.so.6: 
  unknown type [0x13] section `.relr.dyn'
/data/anaconda3/compiler_compat/ld: cannot find /lib/x86_64-linux-gnu/libm.so.6
collect2: error: ld returned 1 exit status
```
- conda 환경의 linker 가 시스템 라이브러리와 호환되지 않음
- PyTorch 가 컴파일된 CUDA 라이브러리를 사용해야 함

- `ensure_cuda_kernels_built()` 가 import 실패 시 바로 False 반환
- LD_LIBRARY_PATH 수정이 반영되기 전에 재시도하지 않음

**수정 후 코드:**
```python
def ensure_cuda_kernels_built() -> bool:
    """
    TurboQuant CUDA 커널 빌드 상태 확인 및 필요시 자동 빌드 시도.
    
    먼저 빌드된 커널 파일이 cuda_kernels/에 있는지 확인하고,
    있으면 바로 True 를 반환합니다. 빌드는 파일이 없는 경우에만 시도합니다.
    
    Returns:
        bool: True if CUDA kernels are available and working, False otherwise
    """
    if not torch.cuda.is_available():
        return False

    # Try importing CUDA kernels
    try:
        from methods.turboquant.turboquant_cuda_kernel import is_cuda_available

        if is_cuda_available():
            return True
    except ImportError as e:
        print(f"[INFO] CUDA kernel import failed: {e}")
        pass

    # Check if pre-built kernels exist in cuda_kernels/
    base_dir = Path(__file__).resolve().parent
    cuda_kernels_dir = base_dir / "turboquant" / "cuda_kernels"

    if cuda_kernels_dir.exists():
        import fnmatch
        required_files = [
            "cuda_qjl_quant.cpython-*.so",
            "cuda_qjl_score.cpython-*.so",
            "cuda_qjl_gqa_score.cpython-*.so",
            "quantization.cpython-*.so",
        ]
        has_all_files = True
        for pattern in required_files:
            if not any(fnmatch.filter(os.listdir(cuda_kernels_dir), pattern)):
                has_all_files = False
                break

        if has_all_files:
            print(f"[INFO] Pre-built CUDA kernels found")
            print(f"[INFO] Attempting reload with updated LD_LIBRARY_PATH")
            
            # Clear module cache to force reload
            if "methods.turboquant.turboquant_cuda_kernel" in sys.modules:
                del sys.modules["methods.turboquant.turboquant_cuda_kernel"]
            
            # Try importing again
            try:
                from methods.turboquant.turboquant_cuda_kernel import is_cuda_available
                if is_cuda_available():
                    return True
            except ImportError as e:
                print(f"[INFO] CUDA kernel import still failed after reload: {e}")
                # Don't attempt to rebuild - let user run build_cuda_kernels.sh manually
                return False
    
    # No pre-built kernels found, or import failed
    return False
```

**변경 사항:**
1. **모듈 캐시 제거**: `del sys.modules["methods.turboquant.turboquant_cuda_kernel"]`
2. **재시도 로직 추가**: LD_LIBRARY_PATH 수정 후 재import 시도
3. **명확한 에러 메시지**: 빌드 시도 없이 수동 빌드 권장

## 추가 고려사항

### 1. 환경 변수 설정
`LD_LIBRARY_PATH` 설정은 Python 스크립트 내부에서 수행되지만,
더 안정적인 방법을 원한다면 `export` 명령어를 사용할 수 있습니다:

```bash
export LD_LIBRARY_PATH=/data/anaconda3/envs/py310/lib/python3.10/site-packages/torch/lib/cuda/lib:/data/anaconda3/envs/py310/lib/python3.10/site-packages/torch/lib:$LD_LIBRARY_PATH
python experiments/turboquant_facade.py
```

### 2. 빌드 문제 해결
만약 빌드 문제가 지속된다면:

```bash
cd methods/turboquant/csrc

# Clean previous build
rm -rf build/

# Set environment variables
export TORCH_CUDA_ARCH_LIST=8.6
export CUDA_PATH=/usr/local/cuda
export CC=/usr/bin/gcc
export CXX=/usr/bin/g++
export LD_LIBRARY_PATH=/data/anaconda3/envs/py310/lib/python3.10/site-packages/torch/lib/cuda/lib:/data/anaconda3/envs/py310/lib/python3.10/site-packages/torch/lib:$LD_LIBRARY_PATH

# Build
python setup.py build_ext --inplace
```

### 3. conda 환경 사용 권장
빌드 문제는 conda 환경의 linker 문제일 수 있습니다:

```bash
# py310 conda 환경 사용
conda activate py310

# 빌드
cd methods/turboquant/csrc
bash build_cuda_kernels.sh
```

1. **LD_LIBRARY_PATH 타이밍 중요**: `import torch` 전에 반드시 설정
2. **모듈 캐시 관리**: `sys.modules` 에서 이전 모듈 제거 후 재import 필요

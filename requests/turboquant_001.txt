# TurboQuant CUDA 커널 빌드 경로 문제 수정

## 문제 설명

### 현재 상황
- `methods/turboquant/csrc/setup.py` 에서 CUDA 커널을 빌드할 때 `.so` 파일을 두 곳에 복사:
  1. `methods/turboquant/cuda_kernels/` (주 목적지)
  2. `methods/turboquant/turboquant/` (import 호환성용)

- `methods/turboquant/turboquant_cuda_kernel.py` 에서 커널을 로드할 때:
  - `_kernel_dir = os.path.join(os.path.dirname(__file__), "cuda_kernels")`
  - 즉, `turboquant_cuda_kernel.py` 와 같은 디렉토리 (`turboquant/`) 에서 `cuda_kernels/` 서브디렉토리를 찾음
  - 결과: `methods/turboquant/turboquant/cuda_kernels/` 를 찾음 (존재하지 않음)

### 문제 증상
1. CUDA 커널 빌드 성공에도 불구하고 `_CUDA_AVAILABLE = False`
2. `experiments/turboquant_facade.py` 실행 시 모든 엔진이 NA/SKIP
3. TSV 파일에서 모든 값이 NA

### 근본 원인
`turboquant_cuda_kernel.py` 의 import 경로와 `setup.py` 의 복사 경로가 불일치:
```
setup.py 복사: methods/turboquant/cuda_kernels/ ✓
turboquant_cuda_kernel.py 검색: methods/turboquant/turboquant/cuda_kernels/ ✗
```

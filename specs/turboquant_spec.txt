## 1. Executive Summary

TurboQuant is Google's online vector quantization algorithm that achieves near-optimal MSE and inner product distortion rates. This implementation extends TurboQuant with three rotation variants (RotorQuant, IsoQuant, PlanarQuant) and adapts it for LLM KV cache compression.

### Key Findings (Critical)

**QJL is NOT suitable for KV Cache despite being part of original TurboQuant design:**

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│  QJL for KV Cache: ❌ CATASTROPHIC FAILURE                                      │
│                                                                                 │
│  Problem: QJL provides unbiased inner products, but softmax exponentially       │
│           amplifies its random noise → catastrophic PPL degradation             │
│                                                                                 │
│  Evidence:                                                                     │
│  - 6 independent teams confirmed softmax variance amplification                │
│  - QJL mode: +300% attention error vs MSE-only                                 │
│  - Generation test: QJL=0/27 pass, MSE-only=18/18 pass                         │
│  - Qwen2.5-3B 3-bit: RefTQ(QJL)=5,128 PPL vs MSE-only=170 PPL                  │
│                                                                                 │
│  Recommendation: Use MSE-only for ALL KV cache operations                       │
│  (QJL only suitable for non-softmax applications: vector search, linear/       │
│   sigmoid attention)                                                            │
└─────────────────────────────────────────────────────────────────────────────────┘
```

## 2. Theoretical Foundations

### 2.1 MSE Distortion Bound

**Theorem** (from paper): TurboQuant achieves near-optimal MSE distortion:

```
D_mse ≤ (√3 × π) / 2 × (1 / 4^b) ≈ 2.7 × Lower Bound
```

Where:
- `b` = bits per coordinate
- `2.7` = constant factor from paper's theoretical analysis
- This is within a small constant factor of information-theoretic lower bound

**Experimental verification** (d=128, unit vectors):

| bits | Theory Bound | RefTQ | RotorQuant | IsoQuant-Fast |
|------|--------------|-------|------------|---------------|
| 2    | 0.170044     | 0.128799 | 0.115858 | 0.116346 |
| 3    | 0.042511     | 0.049446 | 0.034060 | 0.034310 |
| 4    | 0.010628     | 0.019621 | 0.009302 | 0.009431 |

All methods achieve well below theoretical bound.

### 2.2 Inner Product Preservation

**Two-Stage QJL Approach** (from paper):
1. Stage 1: MSE-optimal quantizer with `b-1` bits
2. Stage 2: 1-bit Quantized Johnson-Lindenstrauss (QJL) transform on residual

**QJL Properties**:
- Unbiased inner product estimation
- Low distortion for vector search (no softmax)
- Fails catastrophically with softmax attention

**Why QJL Fails with Softmax**:
```
softmax(x + noise) ≈ exp(x + noise) / Σexp(x + noise)

Variance amplification:
Var[exp(x + noise)] >> Var[exp(x)]

MSE-only advantage:
- Biased reconstruction (scale factor ~2/π for 1-bit)
- BUT: Low variance preserves directional information better
- Result: Better softmax attention scores
```

### 2.3 Information-Theoretic Lower Bounds

Derived from:
- Shannon's distortion-rate function
- Yao's minimax principle

**Lower Bound Formula**:
```
D_min = (π × e / 3) × 2^(-2R)
```
Where `R = b` (rate in bits per dimension)

TurboQuant achieves within factor of `√3 × π / 2 ≈ 2.7` of this bound.

## 3. Rotation Variant Architecture

### 3.1 Overview

| Variant | Rotation Type | Block Size | Parameters (d=128) | FMAs (d=128) | Best Use Case |
|---------|---------------|------------|-------------------|--------------|---------------|
| **TurboQuant** | Dense d×d matmul | d | 16,384 | 16,384 | Baseline reference |
| **RotorQuant** | Clifford Cl(3,0) rotors | 3D | 344 | 2,408 | Legacy compatibility |
| **IsoQuant-Fast** | Quaternion 4D blocks | 4D | 128 | 512 | **Recommended default** |
| **IsoQuant-Full** | Quaternion 4D (q_L v q̄_R) | 4D | 256 | 1,024 | Max quality |
| **PlanarQuant** | 2D Givens rotations | 2D | 128 | 256 | Maximum speed |

### 3.2 IsoQuant (Recommended)

**Mathematical Foundation**:
- Isoclinic decomposition: SO(4) ≅ SU(2) × SU(2)
- Each 4D block treated as quaternion
- Fast mode: `q_L v` (one quaternion multiply)
- Full mode: `q_L v q̄_R` (full quaternion sandwich)

**Advantages**:
- Hardware-aligned 4D blocks (no tail handling)
- 120.5× fewer parameters than TurboQuant
- 5.8× faster than RotorQuant in PyTorch
- ~30µs latency with Triton (memory-bound floor)

### 3.3 PlanarQuant (Fastest)

**Mathematical Foundation**:
- 2D Givens rotations (SO(2))
- Each adjacent coordinate pair rotated by independent angle θ
- Minimum 4 FMAs per pair (theoretical minimum)

**Advantages**:
- Clean 2D alignment (no tail)
- 10–27× faster than RotorQuant in PyTorch
- 2–5× faster than IsoQuant-Fast
- ~30µs with Triton (tied with IsoQuant)

### 3.4 RotorQuant (Legacy)

**Mathematical Foundation**:
- Clifford algebra Cl(3,0) rotors
- Sandwich product: `R v R̃`
- Grade elimination: only odd grades preserved

**Status**:
- 46.6× fewer parameters than TurboQuant
- Kept for backward compatibility
- Outperformed by IsoQuant/PlanarQuant in speed

## 5. KV Cache Optimization

### 5.1 Asymmetric K/V Bit Allocation

**Rationale**:
- Keys: Critical for attention scores (Q @ K^T) → Higher precision
- Values: Used in weighted sum after softmax → Lower precision acceptable

**Default Configuration**:
```python
TurboQuantKVCache(
    d_key=128, d_value=128,
    key_bits=4,  # 3.8x compression
    value_bits=2,  # 4.9x overall compression
    seed=42
)
```

**Memory Calculation** (seq_len=100, d=128):
```
Key indices: 100 × 128 = 12,800 indices
Key bits: 12,800 × 4 = 51,200 bits

Value indices: 100 × 128 = 12,800 indices
Value bits: 12,800 × 2 = 25,600 bits

Total: 76,800 bits (5.33x compression vs FP16)
FP16 baseline: 100 × 256 × 16 = 409,600 bits
```

### 5.2 GQA (Grouped Query Attention) Support

**Runtime Configuration**:
```python
# Detect from model config
gqa_group_size = num_query_heads // num_kv_heads

# Example: Qwen2.5-3B
# num_query_heads = 28, num_kv_heads = 4 → group_size = 7

cache = TurboQuantKVCache(64, 64, bits=3)
keys = torch.randn(1, 4, 10, 64)  # 4 KV heads
values = torch.randn(1, 4, 10, 64)
cache.append(keys, values)

queries = torch.randn(1, 28, 1, 64)  # 28 query heads
scores = cache.attention_scores(queries)  # Auto-expands keys

values_expanded = cache.get_values(group_size=7)  # Expand values
```

## 6. QJL Error Analysis (Research Pending)

### 6.1 Identified Problem

**Symptom**: Catastrophic PPL degradation when using QJL + softmax attention

**Observed Behavior**:
- QJL mode: PPL 5,128 (Qwen2.5-3B, 3-bit)
- MSE-only mode: PPL 170 (same config)
- Pass rate: QJL=0/27, MSE-only=18/18

### 6.2 Hypotheses (To Investigate)

**H1: Softmax Variance Amplification**
- QJL adds random ±1 signs to residual
- softmax amplifies variance exponentially
- Result: Attention scores become noisy

**H2: Implementation Bug**
- QJL matrix S generation might be incorrect
- Residual norm calculation might be wrong
- Inner product estimation formula might be off

**H3: Theoretical Limitation**
- Paper's QJL guarantee assumes no softmax
- Softmax is non-linear, breaks unbiasedness assumption
- This might be inherent to QJL + softmax combination

### 6.3 Investigation Plan

```
Phase 1: Isolate QJL Component
- Implement QJL-only (no MSE quantization)
- Measure attention score variance
- Compare with MSE-only

Phase 2: Measure Variance
- Compute Var[softmax(Q·K̂_QJL)] vs Var[softmax(Q·K̂_MSE)]
- Expected: QJL variance >> MSE variance

Phase 3: Check Paper Implementation
- Compare with back2matching/turboquant reference
- Check QJL sign generation (should be ±1)
- Verify residual norm storage

Phase 4: Test Alternative QJL
- Try different QJL matrix S (orthogonal vs random)
- Try different qjl_dim (full d vs reduced)
- Test QJL on non-softmax attention (linear, gated)
```

### 6.4 Current Recommendation

**Do NOT use QJL for KV cache**. Use MSE-only:
- Lower variance → better softmax stability
- Biased but acceptable for attention
- Proven to work in production (18/18 pass rate)

**QJL is suitable for**:
- Vector search (no softmax)
- Linear attention mechanisms
- Gated attention (no softmax)

## 11. Known Issues & Limitations

1. **QJL + Softmax**: Not compatible, use MSE-only for KV cache
2. **Triton Kernels**: Only implemented for TurboQuant baseline (not RQ/IQ/PQ)
3. **GQA**: Runtime detection works but template specialization would improve performance
4. **CUDA Kernels**: Fallback to PyTorch if custom kernels unavailable
